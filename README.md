# ExamTopics Downloader

Scrapes exam questions from [examtopics.com](https://www.examtopics.com) and generates an **interactive HTML page** for self-study -- complete with answer checking, progress tracking, dark/light theme, and integrated discussion comments.

Supported providers: Amazon AWS, Microsoft, Google, Cisco, CompTIA, ISC2, and any other provider hosted on ExamTopics.

## Features

- **Multiple URL types**: paste a single question, a discussion listing, an exam view, or an exam list URL
- **Search by provider**: use `-p amazon -s SAA-C03` without a full URL
- **Interactive HTML output**: radio-button answers, Peek button, grading, progress bar, dark mode toggle
- **Flexible output formats**: HTML (default), plus optional `.md` or `.txt`
- **Concurrent scraping**: 15 workers, rate-limited at 2 req/s, with retries and backoff
- **Comments support**: optionally include discussion comments in the output (`-c`)
- **Page limit**: limit how many discussion pages to scan (`-n`)

## Requirements

- Python 3.11+
- [`uv`](https://docs.astral.sh/uv/) (Astral's Python package manager)

## Installation

```bash
git clone https://github.com/your-username/examtopics-downloader.git
cd examtopics-downloader
uv sync
```

`uv sync` creates a `.venv/` and installs the package in editable mode along with all runtime dependencies (`requests`, `beautifulsoup4`). After that, run the CLI via `uv run`:

```bash
uv run examtopics --help
```

### Install the CLI globally (no repo needed at runtime)

If you just want the `examtopics` command on your `PATH` (so you can run it from anywhere, without `uv run` or activating a venv), use `uv tool install` from the cloned repo:

```bash
uv tool install .
examtopics --help
```

This builds a dedicated isolated environment for the tool and exposes `examtopics` in `~/.local/bin/` (added to `PATH` automatically on most setups). To upgrade after pulling new changes, re-run `uv tool install --reinstall .`. To uninstall: `uv tool uninstall examtopics-downloader`.

## Usage

### By provider + exam code (most common)

```bash
examtopics -p amazon -s SAA-C03
examtopics -p amazon -s SAA-C03 -n 3 -o my-saa-exam
```

### By URL

```bash
# Single question
examtopics https://www.examtopics.com/discussions/amazon/view/12345-example/

# All discussions for a provider
examtopics https://www.examtopics.com/discussions/amazon/

# Specific exam view
examtopics https://www.examtopics.com/exams/amazon/aws-certified-solutions-architect-associate-saa-c03/view/1/

# Exam listing page
examtopics https://www.examtopics.com/exams/amazon/aws-certified-solutions-architect-associate-saa-c03/
```

### With comments and extra formats

```bash
examtopics -p microsoft -s AZ-104 -c -t md
examtopics -p amazon -s SAA-C03 -o exam_output -t txt
```

## Command-line Options

| Flag | Description |
|------|-------------|
| `url` | (positional) ExamTopics page URL -- discussion, exam view, or exam list. Omit to use `-p`/`-s`. |
| `-p`, `--provider` | Provider slug, e.g. `amazon`, `microsoft`, `google`. Use with `-s`. |
| `-s`, `--search` | Filter discussions by substring (e.g. exam code `SAA-C03`). |
| `-n`, `--pages` | Max discussion listing pages to scan (default: all). |
| `-o`, `--output` | Output file base name (default: exam slug from URL, or `examtopics_output`). |
| `-c`, `--comments` | Include user comments from the discussion thread (default: off). |
| `-t`, `--type` | Extra output format alongside HTML: `md` (keep markdown), `txt` (plain text). Default: `html`. |
| `-h`, `--help` | Show this help message. |

## Output

The primary output is an **interactive HTML file** with:

- Numbered question cards with radio-button answers
- **Peek** -- reveals the correct/popular answer
- **Submit** -- grades all answered questions, shows score and progress bar
- **Comments** -- toggle discussion thread comments per question
- **Source** -- link back to the original ExamTopics page
- Dark/light theme toggle (persisted in `localStorage`)
- Image lightbox for question exhibits and answer areas

If `exam.html` template is found (bundled with the package), full interactive output is produced. If the template is missing, a plain markdown fallback is saved.

## How It Works

1. The tool reads the given URL or searches discussions by provider + keyword
2. It discovers all matching discussion links across paginated listing pages
3. Each question page is scraped concurrently (rate-limited to 2 requests/second)
4. Questions, answer choices, correct answers, images, and comments are extracted
5. An interactive HTML file is generated by merging the data with `exam.html`

## Notes

- **Rate limiting**: 2 requests/second, 15 concurrent workers. Adjust `MAX_CONCURRENT` and `REQUESTS_PER_SEC` in the source if needed.
- **CAPTCHA**: If a CAPTCHA is detected on a page, that question is skipped.
- **Answer extraction**: The tool tries multiple strategies to find the correct answer (explicit "Correct Answer" box, hidden class, voting tally, comment consensus).
- **No warranty**: This is an unofficial tool. Use responsibly and respect ExamTopics' terms of service.
